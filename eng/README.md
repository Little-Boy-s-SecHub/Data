# SecHub/Data — Web & Application Security Curriculum

Vietnamese learning resource collection from beginner to intermediate on Web/Application Security. The repository prioritizes root cause, authorized testing, fix that can be retested, and traceable technical sources.

> [!WARNING]
> Only practice payloads in a local lab or clearly authorized systems. Always cross-check the context, version, and safety conditions noted in each lesson.

## v1 Scope

- 83 lessons belonging to 11 Web/AppSec groups.
- A consolidated cheatsheet file at `security_cheatsheet.md`.
- Not expanded into a full cybersecurity curriculum in v1.
- Obsidian, CTF write-up, HackTricks, and PayloadsAllTheThings are only used to find sources/ideas; not as sources of truth.

## Getting Started

1. Read the [authorized use policy](AUTHORIZED_USE.md).
2. Select lessons according to the `01-...` to `11-...` directory.
3. Use the lessons in the order: objective → foundation → root cause → lab → defense → retest.
4. Only run payloads that match the context/version in the local fixture.

## Content Quality

- Each lesson clearly separates the root cause, attack mechanism, lab conditions, and defense controls.
- Technical claims are attached with reference sources and collected in the links section at the end of the lesson.
- Payloads must include context, conditions, encoding, expected results, and risk level before being used in the lab.
